# 화면 이관 1차 기능 동등성(Parity) 점검표

대상: 저위험(조회/리스트 중심) 페이지

- 기존 Django 템플릿
  - `/frontend/projects`
  - `/frontend/research-notes`
- 신규 Next.js 페이지
  - `/frontend/projects`
  - `/frontend/research-notes`

## 페이지별 기능 동등성 체크리스트

### 1) `/frontend/projects`
- [x] 목록 조회: `/api/v1/projects` 데이터 표시
- [x] 필터: 텍스트 검색(프로젝트명/담당자/기관/ID), 상태(active/inactive)
- [x] 테이블: 핵심 컬럼(이름/담당자/기관/상태/ID) 출력
- [x] 페이징: 클라이언트 페이징(페이지 이동, 총건수)
- [x] 권한 표시: 로그인 세션(`/api/auth/me`) 기준 역할 배지/텍스트 표시
- [x] 에러 처리: API 실패 시 메시지 출력, 빈 결과 안내

### 2) `/frontend/research-notes`
- [x] 목록 조회: `/api/v1/research-notes` 데이터 표시
- [x] 필터: 텍스트 검색(제목/책임자/과제번호/ID)
- [x] 테이블: 핵심 컬럼(제목/책임자/과제번호/기간/파일·연구원)
- [x] 페이징: 클라이언트 페이징(페이지 이동, 총건수)
- [x] 권한 표시: 로그인 세션(`/api/auth/me`) 기준 역할 배지/텍스트 표시
- [x] 에러 처리: API 실패 시 메시지 출력, 빈 결과 안내

## 구/신 화면 기능 parity 표

| 페이지 | 구 화면(기존) | 신 화면(Next.js) | Parity 상태 | 비고 |
|---|---|---|---|---|
| `/frontend/projects` | 카드 목록 중심 조회 | 테이블 기반 조회 화면 | ✅ 동등(조회 기능) | 신 화면은 검색/상태필터/페이징을 추가 제공 |
| `/frontend/research-notes` | 카드 목록 중심 조회 | 테이블 기반 조회 화면 | ✅ 동등(조회 기능) | 신 화면은 검색/페이징을 추가 제공 |

### 항목별 상세 parity (요구 항목)

| 페이지 | 필터 | 테이블 | 페이징 | 권한 표시 | 에러 처리 |
|---|---|---|---|---|---|
| `/frontend/projects` | ✅ 텍스트 + 상태 | ✅ 있음 | ✅ 있음 | ✅ 있음 (`/api/auth/me`) | ✅ 있음 |
| `/frontend/research-notes` | ✅ 텍스트 | ✅ 있음 | ✅ 있음 | ✅ 있음 (`/api/auth/me`) | ✅ 있음 |
